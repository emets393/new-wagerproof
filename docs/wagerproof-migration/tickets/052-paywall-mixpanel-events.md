# Ticket #052 — Paywall Mixpanel events not emitted

**Status:** open
**Filed by:** b08-implementer-2026-05-21
**Filed:** 2026-05-21
**Affects screen / file:** `wagerproof-mobile/services/revenuecat.ts` (trackPaywallResult) → `wagerproof_ios_native/Wagerproof/Features/Paywall/RevenueCatPaywallView.swift`

## What we couldn't ship in scope

RN emits these Mixpanel events from the paywall + subscription flows:
- `paywall_viewed`
- `paywall_dismissed`
- `subscription_started`
- `subscription_purchased`
- `subscription_restored`
- `purchase_failed`
- `purchase_cancelled`

The Swift port doesn't emit any of them yet.

## Why

The native `AnalyticsService` requires a Mixpanel token from `Secrets.swift`, which is generated by `scripts/generate-secrets.sh`. Until B22 wires the bootstrap call from `WagerproofApp.init` with the real token, the analytics service is uninitialized and silently drops events. Adding the call here without B22's secrets pipeline would either crash on missing token or run with a placeholder that pollutes the production project.

## Impact

Marketing dashboards that funnel on `paywall_viewed → subscription_purchased` will see iOS-native users drop out of the funnel. RevenueCat's own dashboard still captures every purchase (it's the source of truth for revenue), so no revenue is lost — only the attribution funnel is impacted.

## Acceptance criteria

- B22 lands the Mixpanel bootstrap with the real `EXPO_PUBLIC_MIXPANEL_TOKEN`.
- The seven events above fire from the corresponding Swift paywall code paths with property shape byte-identical to RN's `analytics.ts`.

## Linked code

- `RevenueCatPaywallView.swift` `.onPurchaseCompleted` / `.onRestoreCompleted` callbacks (no analytics calls yet).
- `SettingsView.swift` `handleSubscriptionPress` / `handleManageSubscriptionTap` (no `paywall_viewed` yet).
- `SecretSettingsView.swift` "Test Paywall" button.

## Notes

This batch deliberately avoids emitting events with the half-initialized `AnalyticsService.shared` because a no-op shared singleton silently swallowing events would hide regressions during B22.
