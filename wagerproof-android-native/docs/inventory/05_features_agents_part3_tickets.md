### `AgentPickTicket.swift` (~791 lines)
- **Purpose & where used**: The boarding-pass "ticket" rendering of a single agent pick. Contains BOTH the compact stacked/list ticket (`AgentPickTicket`) and the full expanded pass (`ExpandedAgentPickTicket`), plus all shared ticket plumbing: route row, team avatar, stamps, status mapping, formatting helpers, and the custom `Shape`s (ticket notch, folder front/back, dash line). Used by: `PickHistoryFolder` (folder peek + rolodex + landscape grid, via `BetItemTicket`), `AgentPickFocusView` (expanded pass), and `OnboardingRevealView` (compact ticket with `teaserBlur: true`).
- **Inputs**: `AgentPickTicket(pick: AgentPick, accent: Color = .appPrimary, teaserBlur: Bool = false)`. Fixed geometry: `height = 250`, `notchY = 150` (shared with rolodex physics — keep in sync). `ExpandedAgentPickTicket(pick:, accent:, onAudit: (() -> Void)? = nil, showsBranding: Bool = false)`; private `notchY = 200`. AgentPick fields used: `matchup`, `gameDate`, `betType`, `pickSelection`, `odds`, `units` (Double), `confidence` (Int, /5), `reasoningText`, `keyFactors: [String]?`, `result` (.won/.lost/.push/.pending), `actualResult`, `sport` (`.sfSymbol`).
- **Layout — compact ticket**:
  1. **Top section (150pt)**: row of [game date, 15pt medium, `appTextPrimary` @0.85] + Spacer + [status badge: 11pt heavy, tracking 0.6, status color text on `status.color.opacity(0.16)` rounded r8]. → `PickRouteLineRow(codeSize: 32)`. → row of away full name / `betType.capitalized` / home full name (12pt, `appTextSecondary`). Bottom overlay: `PickDashLine` dashed `[5,5]` white 0.16 tear line, h-pad 18.
  2. **Bottom stub (100pt)**: stamps row — `PickTicketStamp` Market (leading) / Odds (center) / Units (trailing); then `pickSelection` (15pt heavy, minScale 0.7) + Spacer + confidence chip (`gauge.medium` 9pt + "n/5" 10pt bold, tinted accent).
  - **Cardstock**: `PickTicketShape(notchY:)` — rounded rect r22 continuous with two 9pt-radius circular notches punched (even-odd fill) at the tear line both edges; fill = vertical `LinearGradient(0x141927 → 0x0D101A)`; clipped to r22 so outer notch halves trim; overlay hairline stroke `white.opacity(0.07)` 1pt; shadow `black.opacity(0.22)` r5 y2.
  - **teaserBlur mode** (onboarding): `betType` blurs r5; whole bottom stub blurs r6 + `.clipped()`, overlaid with lock chip — `lock.fill` 11pt + "Unlock in the app" 12pt heavy, accent-tinted, on Capsule fill `0x0D101A.opacity(0.85)` with accent-0.5 1pt stroke.
  - **Expanded pass**: date pill (14pt semibold on white 0.08 Capsule) → `PickRouteLineRow(codeSize: 42)` → names/betType row (13pt) → dashed tear line. Ghosted sport glyph in cardstock: 110pt, white @0.05, top-trailing, hit-testing off. Cardstock gradient `0x151A28 → 0x0D101A`, shadow black 0.6 r18 y-10. Detail grid (spacing 18): rows Market/Units, Odds/Confidence, GameDate/Result (label 13pt secondary, value 17pt semibold monospaced; Units+Confidence tinted accent, Result tinted status color) → "Selection" block (15pt semibold mono) → "SUMMARY" (11pt heavy tracking 1) + reasoning (14pt) → "KEY FACTORS" bulleted with 5pt accent circles → optional audit button (`terminal` icon + "View data audit" 13pt heavy, accent on accent-0.12 r10 with accent-0.35 stroke) → optional `WagerproofTicketFooter`. Bottom pad: 22 branded / 110 unbranded (tail rides into folder pocket).
- **Result-state coloring** (`AgentPick.ticketStatus`): won → "WIN"/`.appWin`; lost → "LOSS"/`.appLoss`; push → "PUSH"/`.appPending`; pending → "PENDING"/`.appTextSecondary`.
- **Sub-views**: `WagerproofTicketFooter` (16pt logo + "Wager"(appTextPrimary@0.7)+"Proof"(0x00E676) 13pt heavy wordmark @0.9); `PickRouteLineRow` (team end = `PickTeamAvatar` disc at codeSize×1.25 stacked above abbreviation code at codeSize×0.8 heavy rounded; route line = 6pt status dot · dashed [3,4] white 0.22 line · sport SF symbol at codeSize×0.42 · dashed line · dot); `PickTeamAvatar` (flat non-glass by design; AsyncImage ESPN logo for MLB else initials at size×0.36 bold white; disc = `appSurfaceElevated` circle + team primary→secondary gradient @0.45 + 1.5pt elevated stroke; `logoContrastPlate` adds opposite-luminance wash when logo ≈ disc color: dark lum<0.45 → white-0.78 @0.15, light lum>0.6 → black @0.55); `PickTicketStamp` (label 12pt secondary over value 16pt semibold mono); `PickTicketFormat` enum (gameDate → "Today"/"Yesterday"/"Tomorrow"/"MMM d, yyyy"/"Pending"; market → Moneyline/Spread/Total/capitalized; units → "%.0fu"/"%.1fu"; splitMatchup on `" @ ", " at ", " vs. ", " vs ", " v. ", " v "`; teamCode → MLB real abbrev via `MLBTeams`, else word initials (max 3) uppercased; teamVisual → MLB gets ESPN logo + brand colors, others neutral `0x1F2937`/`0x6B7280`); Shapes: `PickTicketShape` (notch punch, cornerRadius 22, notchRadius 9), `PickFolderFrontShape` (left brim 26pt below right tab, chamfer at 46% width + 34pt run, top r18, bottom r26), `PickFolderTabShape` (left tab, chamfer down 30pt at 52% width + 34pt), `PickDashLine`.
- **States/interactions**: pure/stateless; parents attach taps. A11y label "matchup, selection, status" + hint "Tap to expand the pick".

### `AgentParlayTicket.swift` (~603 lines)
- **Purpose**: Multi-leg parlay in the same vocabulary but **variable height** (leg rows replace route row; ticket-level Combined Odds/Units replaces per-pick stamps). Deliberately sibling components, not param variants (pick ticket geometry is baked into folder physics). Contains `AgentParlayTicket`, `ExpandedAgentParlayTicket`, `AgentParlayMiniTicket`, and shared `BetItemTicket` switch (`.pick → AgentPickTicket` / `.parlay → AgentParlayTicket`).
- **Geometry**: `headerHeight = 52`, `legRowHeight = 44`, `stubHeight = 100`; `notchY(forLegs:) = header + legRow × clamp(n, 2...4)`; `height = notchY + stub`; shows max 4 legs. Header row [date 15pt + **parlay badge** (`link` 9pt + "N-LEG PARLAY" 10pt heavy tracking 0.6, accent on accent-0.14 Capsule) + status badge] → `ParlayLegRow`s (44pt each, dashed [3,4] white 0.08 dividers) → tear line → stub: stamps Legs / Combined Odds / Units, then "Cross-sport ticket"/"{sport} ticket" (13pt semibold secondary) + confidence chip. Same cardstock as pick.
- `ParlayLegRow`: 6pt leg-status dot · [selection 14pt heavy; sub-row: sport glyph 8pt + `parlayLegMarket(leg)` 10pt + matchup 10pt, secondary] · leg odds 13pt semibold mono.
- **Expanded**: `notchY = 96 + min(legs,4) × 24`. Header: date pill + status badge → `link` 15pt + "N-LEG PARLAY" 19pt heavy rounded tracking 1.5 (accent) → sport subtitle. Ghosted `link` glyph 100pt white 0.05. Detail grid: CombinedOdds/Units, Confidence/Result → "LEGS" numbered list (digit 12pt heavy mono in 18×18 circle tinted legStatusColor on 0.15 fill; selection 14pt semibold mono ≤2 lines + odds; meta "market · matchup · date" 11pt) → SUMMARY → KEY FACTORS → optional footer. **No audit button by design** (parlays lack avatar_picks id). Bottom pad 22/110.
- **Mini**: reuses `AgentPickMiniTicket.width/height/notchY` (178×240, notch 116) + `miniTicketCardstock`. Top: [`link` + "PARLAY" chip accent] + date + `statusCorner` (pending → confidence gauge; graded → status stamp) → up-to-4 one-line legs (4pt status dot + selection 11pt semibold) → dashed [4,4] tear line. Bottom: "N-Leg Ticket" 15pt heavy → `MiniPickStamp` Legs/Odds/Units (units tinted accent) → 2-line `reasoningSnippet` 10pt secondary.
- `parlayLegMarket` free function: ML/Team Total/Spread/Total/Prop/Leg with "F5 "/"1H " period prefix. Status mapping identical to picks.

### `AgentPickFocusView.swift` (~531 lines)
- **Purpose**: Full-screen tap-to-focus + "print-then-swipe" presentation of expanded tickets over darkened backdrop with "Play n/N" header + dots. Presented by `AgentDetailView`/`PublicAgentDetailView` (tap mini ticket, or after generation with `printIntro: true`).
- **Inputs**: `items: [AgentBetItem]`, `accent`, `startIndex = 0` (clamped), `printIntro = false`, `onAudit: (AgentPick) -> Void`, `onClose`. Constants: `cardMaxTiltDeg = 8`, `maxDots = 8` (beyond → text only).
- **Layout**: ZStack — backdrop → VStack[header (opacity 0 until printed) + printerRegion] h-pad 10 → optional high-intensity `PixelWaveBackground` cover (printIntro only) → `topBar` zIndex 10 (back chevron.left + share square.and.arrow.up, 16pt bold white in 42×42 `liquidGlassBackground(Circle)` white-0.16 stroke; share hidden until printed). Backdrops: printIntro → black + inert normal `PixelWaveBackground` (hit-testing off); else `black.opacity(0.74).background(.ultraThinMaterial)`. Header: "Play n/N" 13pt heavy tracking 0.5 secondary `.contentTransition(.numericText())`; dots (6pt, active accent, inactive white 0.25) easeInOut 0.2. Printer region: GeometryReader; `cardW = width - 8`, `topInset = 6`, `.clipped()`. Slot bar: 7pt Capsule vertical gradient `0x2C313C → 0x0E1016`, inner 2pt black-0.7 capsule, top 1pt white-0.14 highlight, shadow black-0.55 r8 y-2.
- **State**: owns `PickTicketMotion` (@Observable): `normalizedRoll`/`normalizedPitch` ∈ [-1,1]; ONE static CMMotionManager app-wide; 60Hz device-motion on serial queue "bet.wagerproof.pick-ticket-tilt"; baseline primed on first sample; `maxRadians = 0.35`, low-pass `smoothing = 0.09`, self-centering `baselineDrift = 0.04`; `live` gate pins output to 0 until `goLive()`. @State: index, appeared, started, printProgress, printed, cardHeight (starts 440, grows monotonically), waveOpacity, shareItem.
- **Animations choreography**:
  1. Present: easeOut 0.25 fade; tap-to-focus also scales 0.9→1.
  2. Print intro: `waveOpacity = 1` instantly → hold 850ms → easeOut 0.5 fade → `runFeed()`.
  3. Receipt feed: 12 chunks, one per 140ms; each: light UIImpactFeedbackGenerator intensity 0.6 + easeOut 0.13 stepping `printProgress += 1/12` (ticket travels from bottom slot to topInset). At chunk 3, pre-warm CoreMotion. After 12: `.success` haptic + spring(0.5, 0.85) flips `printed` (slot fades, header/share appear, hand-off to pager seamless) + `motion.goLive()`.
  4. Accelerometer parallax: current page only — rotation3DEffect(pitch × 8°, x) + (-roll × 8°, y), perspective 0.5.
  5. Close: `motion.stop()`, easeOut 0.2 fade, onClose after 200ms.
- **Interactions**: backdrop tap → close; share → `ImageRenderer` renders current card at width 340, scale 3, `isOpaque = false`, forced dark → ShareSheet (UIActivityViewController). Pager: TabView `.page` hidden index; each page a vertical ScrollView (orthogonal axes; top pad 6, bottom 48); audit enabled on pick pages. Reduce-motion short-circuits (progress=1, printed, no motion).
