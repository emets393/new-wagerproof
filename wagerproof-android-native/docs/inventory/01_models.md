## 2. AgentBetItem.swift — unified pick/parlay feed item (client-only, not serialized)

**`enum AgentBetItem`**: `case pick(AgentPick)` | `case parlay(AgentParlay)` — Kotlin: sealed interface.
- `id`: `"pick_<id>"` / `"parlay_<id>"` (prefix prevents ForEach collision).
- `gameDate`: pick → `pick.gameDate`; parlay → `parlay.displayDate`.
- `createdAt`, `result` (ticket-level roll-up), `units`, `confidence`: pass-through.
- `sportForFilter: AgentSport?`: pick → sport; parlay → `sport.asAgentSport` (nil for multi-sport, which only shows under "All").
- `netUnitsContribution: Double` — **server Formula B parity** (`recalculate_avatar_performance`):
  - pick won → `units * (americanWinMultiplier(odds) ?? 100/110)` (missing/unparsable odds → -110 payout); lost → `-units`; push/pending → 0.
  - parlay won → `units * (settledDecimalOdds - 1)` when `settledDecimalOdds > 1`; else `units * (americanWinMultiplier(combinedOdds) ?? 1.0)` (even money fallback); lost → `-units`; push/pending → 0.
- `static americanWinMultiplier(odds: String?) -> Double?` — strips `+`, parses Int, 0/unparsable → nil; positive → `odds/100`; negative → `100/abs(odds)`.

## 11. DistributionStatistics.swift — pure stats math for agent-stats histograms (client-only, no JSON)

- **`enum StatMetric`**: `winRate`, `netUnits` (rawValues camelCase of case name; UI labels "Win Rate"/"Net Units").
- **`struct NormalFit`**: `mean, sd: Double, count: Int, isEstimated: Bool`. `pdf(x)` = normal density (0 when sd ≤ 0); `share(at:binWidth:)` = pdf × binWidth; `normalizedDensity(at:)` = `exp(-z²/2)` (peak 1; sd≤0 → 1 at mean else 0).
- **`struct DistributionBucket`**: `lower, upper, share: Double`; `mid`, `id = lower`. Share = fraction of cohort (0…1), never a raw count.
- **`struct CurvePoint`**: `x, y: Double`, `id = x`.
- **`enum DistributionStatistics`**:
  - `fit(values, isEstimated) -> NormalFit?`: filter finite; nil for empty; mean = sum/n; sd = 0 when n<2 else sample SD (÷ n−1).
  - `histogram(values, domain, binWidth)`: guard binWidth>0 & domain valid & non-empty finite values; binCount = `max(1, ceil(span/binWidth))`; index = `floor((v-lower)/binWidth)` clamped into [0, binCount-1] (out-of-domain values clamp to edge bins); emit ALL bins (including empty) with share = count/total; last bin's upper capped at domain.upperBound.
  - `curvePoints(fit, domain, binWidth, samples=120)` — evenly-spaced samples of `fit.share`.
  - `normalizedCurvePoints(fit, domain, samples=160)` — samples of normalizedDensity.

## 38. PropsInsight.swift — MLB props widget digest (pure logic)

**`enum MLBPropsInsight`** algorithm (spec §1b):
- `buildPool`: starters get their `pitcher_strikeouts` headline (fallback any pitcher market) as `.starterK`; each lineup batter gets `pickHeadlineProp` and is classified: l10.games ≥ 5 && pct ≥ 70 → hotBat; ≤ 30 → coldBat; else lineupFill. - `teaser`: top pct among signals with l10.games>0; below `leaderFloor` (55) → neutral nil-headline; headline `"<lastName> <marketShort> o<line> · <pct>% L10"`; positive when ≥ 70; smallSample when l10.games < 5.

# Kotlin porting notes

## Serialization strategy

1. **Base Json config** — one shared instance:
   ```kotlin
   val WagerproofJson = Json {
       ignoreUnknownKeys = true      // Supabase rows carry extra columns
       isLenient = true              // helps Num|Str but NOT sufficient alone (see below)
       coerceInputValues = true      // null → default for non-null fields with defaults
       explicitNulls = false
   }
   ```
2. **Num|Str numerics** (Postgres NUMERIC drift). `isLenient` handles `"1.5"` → Double for primitives, but write one explicit reusable serializer to be safe and to also strip `%` where needed:
   ```kotlin
   object FlexDoubleSerializer : KSerializer<Double?> { /* JsonPrimitive: doubleOrNull ?: content.removeSuffix("%").trim().toDoubleOrNull() */ }
   object FlexIntSerializer : KSerializer<Int?> { /* intOrNull ?: doubleOrNull?.toInt() ?: content.toIntOrNull() */ }
   ```
   Apply to: AgentPerformance.netUnits, AgentLeaderboardEntry.netUnits, AgentStatDatum.winRate/netUnits, BinAgent.winRate/netUnits, AgentPick.units, AgentParlay.units, AgentParlayLeg.propLine, TopAgentPickFeedRow.units/agentNetUnits, all MLBF5SplitRow fields, MLBSituationalTrendRow numerics (with `%` strip), MLBModelBreakdownRow.unitsWon/winPct/roiPct, MLBBucketAccuracyRow.winPct/roiPct, SignalPerformance numerics, MLBPlayerPropLineEntry/GameEntry.
3. **Tolerant decode** — Swift's `(try? decode) ?? default` pattern maps to kotlinx **property defaults + `coerceInputValues`** for the missing/null case, but Swift also tolerates *wrong-typed* values (falls to default instead of throwing). For the agent-family models (Agent, AgentPick, AgentParlay, AgentPerformance, AgentDetailSnapshot, etc.) where corrupt prod rows are a known reality, wrap the flexible serializers above OR decode via `JsonElement` intermediates in a repository mapper. Recommended: defaults + Flex serializers cover 95%; enum fallbacks (below) cover the rest.
4. **Enum fallbacks** — Swift decodes unknown enum raw values to a default (`sport → .nfl`, `result → .pending`, `GameType → .unknown`, `AgentParlaySport → .multi`, archetype → nil). kotlinx throws on unknown enum. Options: `Json { decodeEnumsCaseInsensitive }` doesn't help — write a small generic `FallbackEnumSerializer<E>(default)` or use `@JsonNames`. Apply per-field, matching each Swift default exactly. `GameType` also lowercases input first; `CFBFlagConviction` tries raw then uppercased.
5. **Lossy arrays** — AgentPick/AgentParlay lists and parlay `legs`/BinAgent `pending_picks` must skip malformed elements, not fail the batch. Kotlin: decode as `List<JsonElement>` and `mapNotNull { runCatching { json.decodeFromJsonElement<T>(it) }.getOrNull() }`. Centralize as `fun <T> Json.decodeLossyList(...)`.
6. **Dual-casing decoder** (`ArchivedGameData`) — custom `KSerializer` that reads a `JsonObject` and tries `["awayTeam","away_team"]` per field; encode camelCase only.
7. **JSONValue** — do NOT port the enum; use `kotlinx.serialization.json.JsonElement` everywhere Swift uses `JSONValue` (`ai_decision_trace`, `ai_audit_payload`, `raw_game`, `fields`, `data`). Port the JSONValue.swift accessors as JsonElement extension props (`stringValue`, `intValue` accepting int-or-double, subscript, prettyPrinted).
8. **Data-wrapped JSON envelopes** (WagerBotAppComponent.fieldsJSON/rawGameJSON, WagerBotChatGameCard.rawGameJSON, WagerBotChatWidget.dataJSON) — Swift stores re-encoded `Data`; Kotlin should simply hold `JsonElement?` fields serialized under the original keys (`fields`, `raw_game`, `data`).
9. **Dates** — Only `Profile.createdAt` is a `Date`; everything else keeps timestamps as ISO `String` (deliberate — formatting happens in view layer). Recommend keeping String everywhere and `Instant` only for Profile (custom ISO8601 serializer tolerant of missing fractional seconds — Supabase emits both).
10. **camelCase wire formats** — most models are snake_case (`@SerialName` per field), but these are camelCase (no renames needed): CFBTeamReference, CFBDryRunFlag, CFBSignalDefinition, NBAGame, NCAABGame, NCAABTeamMappingEntry, PlaceholderGames types, WidgetDataPayload family, MLBSignalItem, NFLPropRecentGame (`opp/week/actual`), PolymarketPricePoint (`t/p`), NFLTrendSplitCell (`h/l/p/n/pct`), NFLTrendH2HCell, NBAAccuracyBucket/NCAABAccuracyBucket, TeamColors. For NBAGame/NCAABGame confirm how the Android service builds rows — on iOS the store selects/aliases columns before decode.
11. **Swift tuples** → small data classes: `predictedScore (home, away)`, `fullGameRuns (home, away, margin)`, `f5Runs`, `l10Hits (hits, n)`, `miniStrip [(cleared, value)]`, `colors (primary, secondary)`, `combined (w,l,p,units,roi)`.
12. **UInt32 color hexes** → Kotlin `Long`/`Int` (0x22C55E etc.) feeding Compose `Color(0xFF22C55E)`.
13. **@MainActor caches** (CFBTeamAssets, NFLTeamAssets) → singleton objects with thread-safe maps or repository classes injected via DI; the "install once, read sync" pattern maps to a `StateFlow<Map<...>>` or plain `@Volatile var`.
14. **FNV-1a sprite hash** — implement with `ULong` exactly: seed `0xcbf29ce484222325uL`, prime `0x100000001b3uL`, iterate UTF-8 bytes, `(h % 8u).toInt()`. Add a unit test with a known agent id shared with iOS.
15. **Enums switched exhaustively in UI** (AgentSport, SportLeague, tiers) → Kotlin enums with `label`/icon mapping; `sfSymbol` names have no Android meaning — define a parallel icon-resource mapping.
16. **Unit-math lockstep** — UnitsCalculation + AgentBetItem.netUnitsContribution are contracts shared with SQL/web. Port byte-for-byte and add golden tests (−110 default payout, parlay settled-decimal preference, push=0).
