# Agents Feature Inventory — Part 2: Creation Wizard

> Parity contract for the Android (Jetpack Compose) port. Source: `wagerproof-ios-native/Wagerproof/Features/Agents/` (creation wizard + inputs).
> Covers: AgentCreationView, Step1–Step6, AgentCreationGenerationIntroView, AgentBornCelebrationView, Inputs/ (ArchetypeCard, OddsInput, SliderInput, SwipeableEmojiPicker, TimePickerModal, TimezonePickerModal, ToggleInput). ### `AgentCreationView.swift` (~259 lines)
- **Purpose & nav context**: Six-step wizard **host**. Pushed onto the Agents tab nav stack (create-only; edit lives elsewhere). Owns one `@State AgentCreationStore` (fresh draft per entry). Tab bar hidden via `.toolbar(.hidden, for: .tabBar)` — focused full-screen flow.
- **Layout**: body = `stepBody` (a `switch store.step` over the 6 step views, each self-scrolling). Nav bar: inline title from `stepTitles = ["Sport & Style","Identity","Personality","Data & Conditions","Custom Insights","Review"]`; leading toolbar item = `xmark` button (→ discard dialog); trailing = "`{step+1}/6`" text (14pt semibold, `appTextSecondary`); principal = `stepProgressBar`: HStack(spacing 4) of 6 Capsules 22×3, filled `Color(hex:0x00E676)` (brand green) when `idx <= step`, else `appBorder`. Bottom `safeAreaInset`: `navigationFooter` shown only when `step < 5` (Review has its own Create button). Footer = floating Liquid Glass pills, HStack(spacing 12), horizontal padding 16, vertical 6/6: "Back" pill (only when step > 0; 16pt semibold, `appTextPrimary`, minHeight 52, `liquidGlassBackground(in: Capsule, interactive: true)`) + "Next" pill (16pt bold; text black when `canProceed` else `appTextSecondary`; glass tint `0x00E676` at opacity 0.9 enabled / 0.25 disabled; full-width; disabled when `!canProceed`).
- **Stores & bindings**: `@Environment AgentsStore` (reads `agents` → `store.existingAgentNames`, `activeCount`; calls `refresh()` on finish), `@Environment AgentEntitlementsStore` (`isAdmin`, `isPro`, `AgentEntitlementsStore.proMaxActiveAgents` = 10), `@State AgentCreationStore` (reads `step`, `submitState`, `canProceed(from:)`; calls `advance()`, `back()`, `submit(autoModeForcedOff:)`, `loadArchetypesIfNeeded()`). Premium gating: `autoModeForcedOff` = true when NOT admin, IS pro, and `agentsStore.activeCount >= 10` (new agents can't run autopilot when 10 active auto agents exist). `maxLiveAutoAgents` = nil for admin/non-pro, 10 for pro.
- **States**: continue disabled per `store.canProceed(from:)` — step 0: ≥1 sport; step 1: name non-empty trimmed, ≤50 chars, not a case-insensitive duplicate of `existingAgentNames`, emoji non-empty, color non-empty; steps 2–5: always true. Submit failed → alert. `.task` on appear: seeds `existingAgentNames` from `agentsStore.agents` and eagerly loads archetypes (no flash when swiping to Preset). `.onChange(of: agentsStore.agents)` re-syncs names.
- **Interactions**: xmark → `confirmDiscard = true`; Back → `store.back()`; Next → `advance()` (guards on canProceed, then `store.advance()`). Step views own keyboard avoidance (native Forms).
- **Sheets/dialogs/alerts**: (1) `confirmationDialog("Discard Agent?", isPresented: $confirmDiscard, titleVisibility: .visible)` — destructive "Discard" → `dismiss()`, cancel "Keep Editing"; message "Are you sure you want to discard this agent? All progress will be lost." (2) `alert("Couldn't create agent", isPresented: submitFailedBinding)` — OK resets `store.submitState = .idle`; message = failure string from `.failed(msg)`. (3) `fullScreenCover($showGenerationIntro)` → `AgentCreationGenerationIntroView` with completion → hides intro, shows celebration; `.interactiveDismissDisabled(true)`. (4) `fullScreenCover($showCelebration)` → `AgentBornCelebrationView(agent:)` (only if `createdAgent != nil`), also non-dismissable.
- **Animations**: no explicit step-transition animation in host (plain view swap on `store.step`); choreography lives in the two covers.
- **Routes/completion**: `submit()` async → `store.submit(autoModeForcedOff:)` calls `AgentService.create(input: CreateAgentInput)` (edge function). On success sets `createdAgent`, `showGenerationIntro = true`. Intro completes → celebration. Celebration "View Agent" → `finishAndExit`: `await agentsStore.refresh()` then dismiss cover + `dismiss()` back to agents grid. Store detail: submit flips `autoGenerate` off before serializing when `autoModeForcedOff`; name is trimmed before send.
